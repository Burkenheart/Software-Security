# Software-Security

Overview

This repository includes my work from analyzing and improving the security of a client’s software system. For this assignment, I selected the Artemis Financial Vulnerability Assessment Report, which demonstrates my ability to identify security risks and apply secure coding practices. This artifact highlights my understanding of software security, risk mitigation, and the importance of protecting sensitive data.

Reflection

Client Summary and Requirements

The client, Artemis Financial, is a company that required improvements to the security of its software application. Their main concern was protecting sensitive financial data and ensuring secure communication between systems. The goal was to identify vulnerabilities in their existing application and recommend solutions to reduce security risks, such as implementing encryption and secure coding practices.


Strengths and Importance of Secure Coding

One area I did well in was identifying potential vulnerabilities and explaining why they posed a risk to the system. I focused on issues like insecure communication and outdated dependencies, which could expose sensitive data.

Secure coding is important because it helps prevent data breaches, protects user information, and builds trust with customers. Strong software security adds value to a company by reducing the risk of financial loss, legal issues, and damage to its reputation.


Challenges and Helpful Aspects

One of the more challenging parts of the vulnerability assessment was understanding how different security issues could impact the system as a whole. However, this was also one of the most helpful learning experiences because it improved my ability to think critically about security risks instead of just focusing on code functionality.


Increasing Security Layers and Future Approaches

To increase layers of security, I implemented secure communication using encryption and ensured that sensitive data was handled properly. I also reviewed dependencies and removed or updated any that could introduce vulnerabilities.

In the future, I would use tools like dependency checkers, static code analysis tools, and vulnerability scanners to assess risks more efficiently. I would also rely on industry standards and best practices to decide which mitigation techniques to apply.


Ensuring Functionality and Security

To make sure the application remained functional and secure, I tested the code after making changes to confirm that it still worked as expected. After refactoring, I re-ran tests and reviewed the results to ensure that no new vulnerabilities were introduced. This process helped maintain both functionality and security.


Tools, Resources, and Practices

Some of the tools and practices I used included dependency-check tools, secure coding guidelines, and testing frameworks. These resources helped me identify vulnerabilities and verify that my fixes were effective. These are tools and practices I plan to continue using in future projects.


Demonstrating Skills to Employers

From this assignment, I would show employers my ability to perform a vulnerability assessment, identify security risks, and apply secure coding practices. This work demonstrates my understanding of both software development and cybersecurity, as well as my ability to analyze problems and implement effective solutions.
